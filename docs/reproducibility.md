# Reproducibility Guide

## Environment Setup

### Requirements
- Python 3.9 or higher
- pip or conda

### Installation

```bash
# Clone the repo
git clone https://github.com/CarlAmine/EECE_798K_Project.git
cd EECE_798K_Project

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# Install all dependencies
pip install -r requirements.txt
```

### Conda Alternative

```bash
conda create -n eece798k python=3.10
conda activate eece798k
pip install -r requirements.txt
```

## Expected Raw Data Locations

Raw data files are **not committed** to this repository. Place them at the following paths before running data-dependent scripts:

```
data/utah_forge/         ← Utah FORGE p5838 .mat files
  └── p5838_*.mat        ← one or more experiment files

data/lanl/               ← LANL seismic lab data
data/pangaea/            ← PANGAEA lab friction data  
data/fdem_zenodo/        ← FDEM Zenodo dataset
```

See [`datasets.md`](datasets.md) for download sources and file format details.

**Note:** If data is not present, the pipeline wrapper (`scripts/run_final_pipeline.py`) will detect this and print helpful instructions rather than crashing.

## Smoke Test (No Data Required)

The smoke test verifies that all source modules are importable and expected directories exist:


## Running the Final Pipeline

```bash
# Check if data is present and explain what will run
python scripts/run_final_pipeline.py --check

# Run the full final pipeline (requires raw data)
python scripts/run_final_pipeline.py
```

Outputs are written to `results/utah_forge/`.

Final figures are generated by the scripts in the table above. Most figures are already committed in `results/utah_forge/`. To regenerate:

## Known Limitations

1. **Raw data not committed:** Utah FORGE `.mat` files must be obtained separately.
2. **PySINDy version sensitivity:** Some SINDy features depend on the exact PySINDy version. Use `pysindy>=1.7`.
3. **Notebook execution is heavy:** Full notebook runs require data and can take 10–30 minutes per notebook. Precomputed outputs are provided in `results/`.
4. **Exact RSF fitting is slow:** Multi-start optimization can take hours on large datasets.
5. **Platform paths:** Some scripts use hardcoded relative paths. Run from the repository root.
